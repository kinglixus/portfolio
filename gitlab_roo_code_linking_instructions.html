How to Link GitLab Repository to Roo Code on SML Server for Code Modification

This guide provides instructions on how to link your GitLab repository to the Roo Code environment on an SML server, allowing you to modify code and push changes back to GitLab.

Prerequisites

Before you begin, ensure you have the following:

  • GitLab Account: Access to your GitLab repository.
  • Git Client: Installed on the SML server for executing Git commands.
  • SML Server Access: Credentials and access to the SML server where Roo Code is hosted.
  • Roo Code Environment: Access to the Roo Code environment on the SML server.

Steps to Link and Modify Code

1. Clone Your GitLab Repository

First, you need to clone your GitLab repository directly on the SML server using its terminal.

# Open the terminal on your SML server
# Navigate to the directory where you want to clone the repository
cd /path/to/your/desired/directory/on/sml/server

# Clone the repository (replace YOUR_GITLAB_REPO_URL with your actual repository URL)
# You can find the clone URL on your GitLab project page (e.g., HTTPS or SSH)
git clone YOUR_GITLAB_REPO_URL

Example:
If your repository URL is https://gitlab.com/your-username/your-project.git, the command would be:

git clone https://gitlab.com/your-username/your-project.git

Authentication Note: When cloning or pushing, Git Bash may prompt you for your GitLab username and password or a Personal Access Token (PAT). If using HTTPS, it's recommended to use a PAT for security and convenience, especially if you have 2FA enabled.

2. Access Roo Code on SML Server

Once the repository is cloned, navigate to your project directory within the Roo Code environment.

Navigate to your project directory within the Roo Code environment.

# Example: Navigate to your project directory within Roo Code's terminal
cd /path/to/your/project/on/sml/server/your-project-name

3. Make Code Modifications

Now you can make changes to your code within the Roo Code environment.

Sample Code Modification (e.g., in a JavaScript file script.js):

Original script.js:

console.log("Hello, World!");

Modified script.js:

console.log("Hello, Roo Code!");
// Added a new feature
function greet(name) {
    console.log("Greetings, " + name + "!");
}
greet("User");

4. Stage, Commit, and Push Changes to GitLab

After making your modifications in Roo Code, use Git Bash (or the terminal within Roo Code if it supports Git) to stage, commit, and push your changes back to your GitLab repository.

# Navigate to your project directory (if not already there)
cd /path/to/your/project/on/sml/server/your-project-name

# Check the status of your changes
git status

# Stage your changes (add all modified files)
git add .

# Commit your changes with a descriptive message
git commit -m "feat: Add greeting function and update message"

# Push your changes to the remote repository (e.g., 'origin' is the default remote, 'main' or 'master' is the branch name)
git push origin main

Note: Replace main with your actual branch name if it's different (e.g., master, develop, or a feature branch).

5. Verify Changes on GitLab

Go to your GitLab repository in your web browser to confirm that your changes have been successfully pushed. You should see your new commit and the updated files.

By following these steps, you can effectively link your GitLab repository to the Roo Code environment on your SML server for seamless code modification and version control.
