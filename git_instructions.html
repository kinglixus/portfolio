Connecting Your GitLab Project to Roo Code

This guide explains how to bring your existing GitLab repository into Roo Code and make changes.

Step 1: Clone Your GitLab Repository in Roo Code

First, you need to clone your GitLab repository into your local workspace where Roo Code can access it. Roo Code operates within your local file system, so you'll use standard Git commands.

  1. Open a Terminal in Roo Code: If you're interacting with Roo Code, you can request to execute a command in a new terminal.
  2. Navigate to your desired directory: Use the cd command to go to the directory where you want to clone your project. For example, if you want it in a new folder called my-gitlab-project within your current workspace: 
    mkdir my-gitlab-project
cd my-gitlab-project

    Example:

    C:\Users\YourUser\Projects> mkdir my-gitlab-project
C:\Users\YourUser\Projects> cd my-gitlab-project
C:\Users\YourUser\Projects\my-gitlab-project>
  3. Clone the GitLab Repository: Get the HTTPS or SSH URL for your GitLab repository. 
    git clone  .

    Replace with the actual URL of your GitLab repository. The . at the end clones into the current directory.

    Example:

    C:\Users\YourUser\Projects\my-gitlab-project> git clone https://gitlab.com/your-username/your-repo.git .

Step 2: Commit Your Changes and Push to GitLab

After making changes in Roo Code, you'll want to commit them and push them back to your GitLab repository.

  1. Check for Changes: 
    git status

    Example:

    C:\Users\YourUser\Projects\my-gitlab-project> git status
On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  (use "git add ..." to update what will be committed)
  (use "git restore ..." to discard changes in working directory)
        modified:   index.html

no changes added to commit (use "git add" and/or "git commit -a")
  2. Stage Changes: 
    git add .

    Example:

    C:\Users\YourUser\Projects\my-gitlab-project> git add .
  3. Commit Changes: 
    git commit -m "Update project in Roo Code"

    Use a meaningful commit message.

    Example:

    C:\Users\YourUser\Projects\my-gitlab-project> git commit -m "Update project in Roo Code"
[main 8765432] Update project in Roo Code
 1 file changed, 2 insertions(+), 1 deletion(-)
  4. Push to GitLab: Push your local branch (e.g., main or master) to your GitLab remote (usually named origin). 
    git push origin main

    Replace main with your actual branch name if it's different.

    Example:

    C:\Users\YourUser\Projects\my-gitlab-project> git push origin main

Important Notes:

  • You will likely be prompted for your Git credentials (username and password/personal access token) for GitLab during these operations.
  • Roo Code provides a terminal environment, so all standard Git commands are available.
